# Construction Review Engine — Future Vision Backlog

## Product discipline

The current product goal is:

> Upload one construction document and receive a faster, clearer,
> more useful review than the user could reasonably produce alone.

Items in this backlog are preserved for future development but are not
automatically current priorities.

---

## Phase 1 — Indispensable Document Review

- Automatic AI review
- Document Q&A
- Search
- Verified page citations
- Risk extraction
- Metadata extraction
- Field-order analysis
- Review export
- Review history
- User corrections and feedback
- Document-type-specific review workflows

- Project-wide document search

- RFI-to-drawing relationships
- RFI-to-specification relationships
- Field-order duplication detection

- Document relationship graph

## Ideas deliberately deferred

New ideas belong here until they are selected through the product
prioritization process.

A future capability should not interrupt the current milestone merely
because it is technically possible or strategically interesting.
